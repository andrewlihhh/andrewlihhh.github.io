    配置虚拟主机

    Apache中，虚拟主机的搭建有两种方式

    1. 在主配置文件中搭建：需要手动开启虚拟主机（基于域名）NamevirtualHost *:80

    2. 在专门的虚拟主机文件中配置

      1. 在主配置文件中加载虚拟主机配置文件、

      2. 虚拟主机配置文件中配置虚拟主机

        extra

        1. 增加站点文件夹 ：documentRoot

        2. 增加主机名字，文件夹别名：ServerName

        3. 增加DNS：C:\Windows\System32\drivers\etc/hosts文件

        4. 需要给当前设定的站点（目录）访问权限 ，<Directory ‘站点目录’>

          ​ Order Deny,Allow

          ​ Deny from

          ​ Allow from

        5. 配置访问列表：当访问是一个目录时（没有指定具体要访问的文件）

          Options Indexes[FollowSymLinks]，如果访问目录，列表显示所有文件

        6. 当用户访问指定目录不指定文件时，通常需要系统指定一个默认文件。

          DirextoryIndex默认的文件列表，使用空格分离。

          注意：如果网站中整个网站根目录允许访问呢，当时其中还有其他文件夹不允许访问：可以增加多个Directory标签，针对不同的文件夹

      3. 一旦启用虚拟主机配置文件，那么虚拟主机地址localhost就不再生效，如果想要生效：为localhost增加单独的虚拟主机

        具体操作：再hotds里面增加一个VirtualHost标签

    Isset（）：判断数据存储的变量本身是否存在，存在变量返回true，否则返回false

    计算机码

    计算机码：原码、反码、补码

    原码：二进制，第一位为符号位

    反码：针对负数：符号位不变，其他位取反

    补码：针对负数：反码加一，二进制相加

    计算机中存的都是补码

    系统中存在两个0：+0和-0
    +0： 00000000

    -0： 10000000 原码

    取反： 11111111

    补码： 00000000

    最终在计算机中的存储也是一样的。

    位运算符

    位运算：取出计算机中存储的数，进行操作，由于计算机中存储的补码，因此位运算即为补码运算

    &

    |

    ~

    ^

    <<：按位左移，右边补0

    />/>：按位右移，正数补0，负数补1

    注意：

    1. 系统进行任何位运算的时候都使用补码
    2. 运算结束后，必须将运算得到的补码转换成原码才是最终要显示的数据。如果符号位是0，则为正数，如果符号位是1，则是负数。

    按位左移：乘以2的操作

    按位右移：除以2的操作（不完全正确）：整数除2不会出现小数

    文件包含

    文件包含：在一个php脚本中，将另外一个文件（php）包含进来：去合作完成一件事情

    文件包含的作用

    1. 要么使用被包含文件的内容，实现代码的共享，重用。向上包含（索要）。再当前脚本要用其他代码之前包含别的文件
    2. 要么自己有东西可以给其他文件使用，实现代码的重用：向下包含（给予）。。在自己有某个东西时，需要别的脚本来显示

    最大的作用：分工写作，每个脚本做到事情不一样，因此可以使用协作方式，让多个脚本共同完成一件事情。

    文件包含四种形式

    Include：

    Include_once：系统自动判断文件包含过程中，是否已经包含过。

    Require：与include相同

    Require_once：

    包含基本语法：

    include ‘文件路径/文件名字’；

    include （‘文件路径/文件名字’）；

    向下包含：先准备内容，然后包含另外的文件，在另外的文件中打印显示。注意运行的脚本是包含文件的脚本。

    文件加载原理

    php代码的执行流程：

    1. 读取代码文件，即程序
    2. 编译：将php代码转换成字节码，生成pocode
    3. zendengine来解析opcode，按照字节码去进行逻辑运算
    4. 转换成对应的html代码

    文件加载原理：

    1. 在文件加载的时候，系统会自动地将包含文件中的代码嵌入到当前文件中
    2. 加载位置：在哪加载，对应的文件中的代码嵌入的位置就是对应的include位置
    3. php中被包含的文件是单独进行编译的

    如何体现呢：php文件在编译的过程中如果出现了语法错误，那么会执行失败（不会执行）。但是如果是被包含文件有错误是，系统会在执行到包含include这条语句的时候才报错。

    Include 和require

    如果是include的话，在一个文件中include同一个文件两次可能会出错，因此使用include_once会能增加健壮性

    require和include的区别在于报错的形式不一样，include会报warning，不会影响代码运行。

    require会报fatal error，直接终止程序。

    文件加载路径

    绝对路径：

    1. 从磁盘的根目录开始（本地绝对路径）

      windows：盘符：/路径php文件

      linux：/路径/php文件

    2. 从网站的根目录开始（网络绝对路径）

      /：相对网站主机名字对应的路径

    相对路径：从当前文件所在目录开始的路径

    ​ .|./，虽然在windows在可以使用反斜杠，但是推荐使用正斜杠。.表示当前文件夹，./表示当前文件夹的路径

    ​ ../

    绝对路径和相对路径的加载区别

    1. 绝对路径相对效率偏低，但是相对安全
    2. 相对路径效率较高，但相对不安全

    文件嵌套包含

    文件嵌套包含：一个文件包含两外一个文件，同时被包含的文件又包含一个文件

    嵌套包含的时候，就很容易出现相对路径出错的问题，相对路径会因为文件包含出错。

    只有在使用相对路径即.|./的时候才可能出错，不使用相对路径的表示方法就一般 不会出错。

    函数定义语法

    关键点：

    1. function关键字
    2. 函数名
    3. 参数，形参实参
    4. 函数体
    5. 返回值

    Function 函数名（参数）{

    ​ //函数体

    ​ //返回值：

    }

    定义函数的目的：实现一个功能，因此一个函数一个功能。

    函数实在代码执行阶段，碰到函数名字的时候才会调用，不是在编译阶段

    函数的调用特点：只要系统在内存中能够找到对应的函数，就可以执行，函数调用可以在函数定义之前。

    函数执行的内存分析：

    1. 读取代码进入代码段（编译，将代码变成字节码存储到内存
    2. 根据代码逐行执行

    以上原因：编译和执行是分开的，先编译后执行，只要执行的时候函数已经编译进入内存，执行的时候就可以调用
